## Методические указания

### Теоретическая часть

#### Что такое RAID?
**RAID (Redundant Array of Independent Disks)** - технология объединения нескольких физических дисков в логический блок для повышения производительности, надежности или обоих параметров.

#### Типы RAID

| Тип RAID | Описание | Преимущества | Недостатки | Минимальное количество дисков |
|----------|----------|--------------|------------|-------------------------------|
| **RAID 0** | Чередование (striping) | Высокая производительность | Нет отказоустойчивости | 2 |
| **RAID 1** | Зеркалирование (mirroring) | Высокая надежность, быстрое чтение | Низкое использование дискового пространства | 2 |
| **RAID 5** | Чередование с распределенной четностью | Баланс производительности и надежности | Сложнее в восстановлении | 3 |
| **RAID 6** | Чередование с двойной четностью | Высокая отказоустойчивость | Высокие накладные расходы | 4 |
| **RAID 10** | Комбинация RAID 1 и RAID 0 | Высокая производительность и надежность | Высокая стоимость | 4 |

#### Преимущества программного RAID
- **Независимость от оборудования** - работает на любом оборудовании
- **Гибкость настройки** - можно создавать массивы любой конфигурации
- **Интеграция с Linux** - полная интеграция с ядром Linux
- **Поддержка всех типов RAID** - mdadm поддерживает все основные типы RAID

#### Инструменты для работы с RAID
- **mdadm** - основной инструмент для управления программными RAID в Linux
- **cat /proc/mdstat** - просмотр состояния RAID-массивов
- **lsblk** - отображение информации о блочных устройствах

### Практическая часть

Данная работа выполняется на ВМ server.

#### Шаг 2: Создание RAID 1 из чистых дисков

**Подготовка дисков `/dev/vdc` и `/dev/vdd`:**
```bash
# Проверка, что диски не используются
umount /dev/vdc 2>/dev/null || true
umount /dev/vdd 2>/dev/null || true

# Очистка суперблоков (если есть)
mdadm --zero-superblock /dev/vdc /dev/vdd 2>/dev/null || true
```

**Создание RAID 1:**
```bash
# Создание RAID 1 из двух дисков
mdadm --create --verbose /dev/md0 --level=1 --raid-devices=2 /dev/vdc /dev/vdd

# Проверка состояния массива
cat /proc/mdstat
mdadm --detail /dev/md0
```

**Ожидаемый вывод:**
```
Personalities : [raid1] 
md0 : active raid1 vdd[1] vdc[0]
      1047552 blocks super 1.2 [2/2] [UU]
      bitmap: 0/1 pages [0KB], 65536KB chunk

unused devices: <none>
```

#### Шаг 3: Форматирование и монтирование RAID 1

**Форматирование массива:**
```bash
# Форматирование в ext4
mkfs.ext4 /dev/md0

# Добавление метки
e2label /dev/md0 RAID1_DATA
```

**Создание точки монтирования:**
```bash
# Создание каталога
mkdir -p /mnt/raid1

# Установка прав доступа
chmod 755 /mnt/raid1
```

**Временное монтирование:**
```bash
# Монтирование массива
mount /dev/md0 /mnt/raid1

# Проверка монтирования
df -h
mount | grep raid1
```

#### Шаг 4: Демонстрация отказоустойчивости RAID 1

**Создание тестовых файлов:**
```bash
# Создание тестовых файлов
echo "Тестовый файл RAID 1 - данные до отказа" |   tee /mnt/raid1/test_before_failure.txt
echo "Второй тестовый файл" |   tee /mnt/raid1/test_file2.txt

# Проверка создания файлов
ls -la /mnt/raid1/
```

**Имитация отказа диска:**
```bash
# Помечаем диск как вышедший из строя
mdadm --fail /dev/md0 /dev/vdd

# Удаляем диск из массива
mdadm --remove /dev/md0 /dev/vdd

# Проверка состояния массива
cat /proc/mdstat
mdadm --detail /dev/md0
```

**Проверка работоспособности:**
```bash
# Проверка, что массив продолжает работать
df -h
cat /mnt/raid1/test_before_failure.txt

# Проверка состояния файлов
ls -la /mnt/raid1/
```

**"Возврат" диска и восстановление:**
```bash
# Очистка диска (имитация подключения нового диска)
mdadm --zero-superblock /dev/vdd
wipefs -a /dev/vdd

# Добавление диска обратно в массив
mdadm --add /dev/md0 /dev/vdd

# Наблюдение за процессом восстановления
watch cat /proc/mdstat
```

**Ожидаемый процесс восстановления:**
```

      1047552 blocks super 1.2 [2/1] [U_]
      [=>...................]  recovery =  5.0% (53248/1047552) finish=0.5min speed=30427K/sec
      bitmap: 1 pages [4KB], 65536KB chunk

```

#### Шаг 5: Создание RAID 5 из всех трех дисков

**Подготовка всех дисков:**
```bash
# Размонтирование RAID 1
umount /mnt/raid1

# Остановка RAID 1
mdadm --stop /dev/md0

# Очистка всех дисков
mdadm --zero-superblock /dev/vdc /dev/vdd
mdadm --zero-superblock /dev/vdb

# Очистка разделов на /dev/vdb (если нужно)
umount /mnt/data_ext4 /mnt/data_xfs /mnt/data_vfat 2>/dev/null || true
wipefs -a /dev/vdb
```

**Создание RAID 5:**
```bash
# Создание RAID 5 из трех дисков
mdadm --create --verbose /dev/md1 --level=5 --raid-devices=3 /dev/vdb /dev/vdc /dev/vdd

# Проверка состояния массива
cat /proc/mdstat
mdadm --detail /dev/md1
```

**Ожидаемый вывод:**
```
Personalities : [raid6] [raid5] [raid4] 
md1 : active raid5 vdd[3] vdc[1] vdb[0]
      2095104 blocks super 1.2 level 5, 512k chunk, algorithm 2 [3/3] [UUU]
      bitmap: 0/1 pages [0KB], 65536KB chunk

unused devices: <none>
```

#### Шаг 7: Постоянное монтирование RAID-массивов

**Резервное копирование оригинального файла:**
```bash
# Резервное копирование fstab
cp /etc/fstab /etc/fstab.backup
```

**Добавление записей в /etc/fstab:**
```bash
# Получение UUID массивов
blkid /dev/md1

# Добавление записей в fstab (если RAID 1 активен)
echo 'UUID=ваш-uuid-raid5 /mnt/raid5 ext4 defaults 0 2' |   tee -a /etc/fstab
```

**Проверка конфигурации:**
```bash
# Проверка синтаксиса
mount -a

# Проверка успешного монтирования
df -h
mount | grep raid
```

#### Шаг 8: Мониторинг и обслуживание RAID-массивов

**Проверка состояния массивов:**
```bash
# Просмотр состояния всех массивов
cat /proc/mdstat

# Детальная информация о массиве
mdadm --detail /dev/md1

# Проверка суперблоков
mdadm --examine /dev/md1
```

**Настройка мониторинга:**
```bash
# Сохранение конфигурации
mdadm --detail --scan |   tee -a /etc/mdadm/mdadm.conf

# Обновление initramfs
update-initramfs -u
```

**Проверка производительности:**
```bash
# Проверка производительности дисков
iostat -x 1 5

# Проверка использования массива
iotop -a -o
```

## Типичные проблемы и их решения

### Проблема 1: "mdadm: Cannot get array info for /dev/md0"
**Причина:** Массив не создан или не активен
**Решение:**
```bash
# Проверить состояние массивов
cat /proc/mdstat
# Пересоздать массив при необходимости
```

### Проблема 2: "mdadm: Device or resource busy"
**Причина:** Диск используется другой программой
**Решение:**
```bash
# Проверить использование диска
lsof +D /путь/к/диску
# Размонтировать диск
umount /dev/диск
```

### Проблема 3: "mdadm: No arrays found in config file or automatically"
**Причина:** Конфигурация не сохранена
**Решение:**
```bash
# Сохранить конфигурацию
mdadm --detail --scan |   tee -a /etc/mdadm/mdadm.conf
# Обновить initramfs
update-initramfs -u
```

### Проблема 4: Массив не запускается при загрузке
**Причина:** Неправильная конфигурация
**Решение:**
```bash
# Проверить конфигурацию
cat /etc/mdadm/mdadm.conf
# Проверить fstab
cat /etc/fstab
```

### Проблема 5: Ошибка при добавлении диска
**Причина:** Диск не очищен или поврежден
**Решение:**
```bash
# Очистить диск
mdadm --zero-superblock /dev/диск
wipefs -a /dev/диск
```

## Рекомендации по безопасности

1. **Резервное копирование данных**
   - Всегда создавайте резервные копии перед работой с RAID
   - Используйте `dd` для создания образов дисков

2. **Проверка команд**
   - Внимательно проверяйте команды перед выполнением
   - Убедитесь, что указываете правильные устройства

3. **Работа с правами root**
   - Выполняйте команды с минимально необходимыми привилегиями
   - Не используйте ` ` без необходимости

4. **Мониторинг системы**
   - Регулярно проверяйте состояние RAID-массивов

   - Настройте email-оповещения о проблемах с RAID
   - Используйте `mdadm --monitor` для постоянного контроля
