## Методические указания

### Теоретическая часть

#### Что такое Samba?
**Samba** - это программное обеспечение, реализующее протоколы SMB/CIFS для обеспечения совместимости между Unix-системами и Windows.

#### Архитектура Samba

| Компонент | Описание | Назначение |
|-----------|----------|------------|
| **smbd** | Основной демон | Обслуживание файловых и принтерных ресурсов |
| **nmbd** | Демон NetBIOS | Обработка имен и служба WINS |
| **smbclient** | Клиентская утилита | Подключение к SMB-ресурсам |
| **testparm** | Проверка конфигурации | Валидация конфигурационного файла |

#### Преимущества Samba
- **Кроссплатформенность** - работает с Windows, Linux, macOS
- **Гибкость** - поддержка различных методов аутентификации
- **Безопасность** - поддержка шифрования и различных уровней доступа
- **Интеграция** - возможность интеграции с Active Directory

#### Типы доступа к ресурсам
- **Публичный доступ** - без аутентификации
- **Приватный доступ** - с аутентификацией пользователей
- **Гостевой доступ** - ограниченный доступ для гостей

### Практическая часть

#### Шаг 1: Установка Samba

**Установка пакетов:**
```bash
# Обновление системы
sudo apt-get update

# Установка Samba
sudo apt-get install samba samba-common-bin

# Проверка установки
sudo systemctl status smbd
sudo systemctl status nmbd
```

**Резервное копирование конфигурации:**
```bash
# Резервное копирование оригинального файла
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.backup
```

#### Шаг 2: Базовая настройка Samba

**Создание каталогов для общих ресурсов:**
```bash
# Создание каталогов
sudo mkdir -p /srv/samba/public
sudo mkdir -p /srv/samba/private
sudo mkdir -p /srv/samba/shared

# Установка прав доступа
sudo chmod 755 /srv/samba/public
sudo chmod 770 /srv/samba/private
sudo chmod 775 /srv/samba/shared

# Установка владельца
sudo chown root:sambashare /srv/samba/private
sudo chown root:sambashare /srv/samba/shared
```

**Настройка конфигурации:**
```bash
# Редактирование конфигурационного файла
sudo nano /etc/samba/smb.conf
```

**Пример конфигурации:**
```ini
[global]
   workgroup = WORKGROUP
   server string = Samba Server %v
   netbios name = ubuntu
   security = user
   map to guest = bad user
   dns proxy = no

[public]
   path = /srv/samba/public
   browsable = yes
   writable = yes
   guest ok = yes
   read only = no

[private]
   path = /srv/samba/private
   browsable = yes
   writable = yes
   guest ok = no
   read only = no
   valid users = @sambashare

[shared]
   path = /srv/samba/shared
   browsable = yes
   writable = yes
   guest ok = no
   read only = no
   valid users = @sambashare
```

#### Шаг 3: Создание пользователей Samba

**Создание группы для пользователей:**
```bash
# Создание группы
sudo groupadd sambashare
```

**Создание пользователей системы:**
```bash
# Создание пользователей
sudo useradd -M -G sambashare user1
sudo useradd -M -G sambashare user2

# Установка паролей
sudo passwd user1
sudo passwd user2
```

**Создание пользователей Samba:**
```bash
# Добавление пользователей в Samba
sudo smbpasswd -a user1
sudo smbpasswd -a user2

# Включение пользователей
sudo smbpasswd -e user1
sudo smbpasswd -e user2
```

#### Шаг 4: Настройка прав доступа

**Настройка прав на каталоги:**
```bash
# Установка ACL для гибкого управления правами
sudo setfacl -m u:user1:rwx /srv/samba/shared
sudo setfacl -m u:user2:rwx /srv/samba/shared

# Проверка прав
sudo getfacl /srv/samba/shared
```

**Настройка специальных прав:**
```bash
# Создание каталога с правами на создание, но без удаления
sudo mkdir /srv/samba/readonly_create
sudo chmod 755 /srv/samba/readonly_create
sudo chown root:sambashare /srv/samba/readonly_create

# Настройка в конфигурации
# add to smb.conf:
# [readonly_create]
#    path = /srv/samba/readonly_create
#    browsable = yes
#    writable = yes
#    guest ok = no
#    read only = no
#    create mask = 0644
#    directory mask = 0755
```

#### Шаг 5: Запуск и тестирование Samba

**Запуск служб:**
```bash
# Перезагрузка конфигурации
sudo systemctl restart smbd
sudo systemctl restart nmbd

# Включение автозапуска
sudo systemctl enable smbd
sudo systemctl enable nmbd
```

**Проверка конфигурации:**
```bash
# Проверка синтаксиса
sudo testparm

# Проверка доступных ресурсов
sudo smbclient -L localhost -U%
```

#### Шаг 6: Тестирование подключения

**Тестирование из Linux:**
```bash
# Установка клиента (если не установлен)
sudo apt-get install smbclient

# Подключение к ресурсу
sudo smbclient //localhost/public -U user1

# Проверка доступа к приватному ресурсу
sudo smbclient //localhost/private -U user1
```

**Тестирование из Windows:**
```bash
# Проверка доступности сервера
ping имя_сервера

# Просмотр доступных ресурсов
net view \\имя_сервера
```

#### Шаг 7: Безопасность и мониторинг

**Настройка брандмауэра:**
```bash
# Открытие портов для Samba
sudo ufw allow samba

# Или вручную
sudo ufw allow 139/tcp
sudo ufw allow 445/tcp
sudo ufw allow 137/udp
sudo ufw allow 138/udp
```

**Настройка логирования:**
```bash
# Редактирование конфигурации для логирования
# add to smb.conf:
# [global]
#    log file = /var/log/samba/%m.log
#    log level = 2
#    max log size = 1000
```

**Мониторинг активности:**
```bash
# Просмотр активных подключений
sudo smbstatus

# Просмотр логов
sudo tail -f /var/log/samba/log.smbd
```

## Типичные проблемы и их решения

### Проблема 1: "Connection refused" при подключении
**Причина:** Служба Samba не запущена или брандмауэр блокирует порты
**Решение:**
```bash
# Проверить статус служб
sudo systemctl status smbd
sudo systemctl status nmbd

# Проверить брандмауэр
sudo ufw status
```

### Проблема 2: "NT_STATUS_LOGON_FAILURE"
**Причина:** Неправильный пароль или пользователь не существует
**Решение:**
```bash
# Проверить пользователей
sudo pdbedit -L

# Пересоздать пользователя
sudo smbpasswd -a username
```

### Проблема 3: "Permission denied" при доступе к ресурсу
**Причина:** Неправильные права на каталог или файл
**Решение:**
```bash
# Проверить права
ls -la /path/to/resource

# Установить правильные права
sudo chmod 755 /path/to/resource
sudo chown user:group /path/to/resource
```

### Проблема 4: Ресурс не виден в сети
**Причина:** Неправильная конфигурация или проблемы с NetBIOS
**Решение:**
```bash
# Проверить конфигурацию
sudo testparm

# Проверить NetBIOS имя
sudo nmblookup -M --
```

### Проблема 5: Ошибки при копировании файлов
**Причина:** Недостаточно прав или проблемы с файловой системой
**Решение:**
```bash
# Проверить права на запись
sudo -u username touch /path/to/resource/test

# Проверить файловую систему
df -h /path/to/resource
```

## Рекомендации по безопасности

1. **Аутентификация**
   - Используйте сильные пароли
   - Регулярно меняйте пароли пользователей
   - Ограничивайте количество попыток входа

2. **Права доступа**
   - Принцип минимальных привилегий
   - Регулярно проверяйте права доступа
   - Используйте ACL для гибкого управления

3. **Брандмауэр**
   - Ограничьте доступ только необходимым IP-адресам
   - Регулярно проверяйте правила брандмауэра
   - Используйте VPN для удаленного доступа

4. **Логирование**
   - Включите детальное логирование
   - Регулярно просматривайте логи
   - Настройте оповещения о подозрительной активности

5. **Обновления**
   - Регулярно обновляйте Samba
   - Следите за security advisories
   - Тестируйте обновления перед установкой
