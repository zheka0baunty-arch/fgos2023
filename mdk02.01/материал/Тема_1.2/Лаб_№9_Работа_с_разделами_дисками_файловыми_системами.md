# Лабораторное занятие №9. Работа с разделами, дисками и файловыми системами

## Объем
2 часа

## Цель
Освоить работу с дисками, разделами и монтирование файловых систем в Linux. Научиться эффективно управлять дисковым пространством, создавать и форматировать разделы, а также настраивать постоянное монтирование файловых систем.

## Задачи

2. Создать разделы на дополнительном диске (/dev/vdb) с использованием parted

5. Проверить работоспособность созданных файловых систем
6. Освоить управление точками монтирования через /etc/fstab

## Методические указания

### Теоретическая часть

#### Файловые системы в ALT Linux
ALT Linux поддерживает несколько типов файловых систем, каждая из которых имеет свои особенности:

| Тип файловой системы | Описание | Преимущества | Недостатки | Рекомендуемое использование |
|---------------------|----------|--------------|------------|---------------------------|
| **ext4** | Расширенная файловая система, развитие ext3 | Высокая надежность, поддержка больших файлов, journaling | Меньшая производительность по сравнению с XFS | Системные диски, корневые разделы |
| **xfs** | Файловая система высокой производительности | Отличная производительность при работе с большими файлами, эффективное использование дискового пространства | Сложнее в восстановлении, требует больше оперативной памяти | Серверы, базы данных, медиафайлы |
| **vfat** | Файловая система совместимости с Windows | Полная совместимость с Windows, поддержка всех ОС | Ограничения на размер файлов (4 ГБ), нет поддержки прав доступа | Обмен данными между системами, загрузочные флешки |
| **btrfs** | Современная файловая система с продвинутыми функциями | Снимки, сжатие, RAID-функции, самовосстановление | Менее стабильна, требует тщательного тестирования | Экспериментальные системы, резервное копирование |

#### Основные понятия
- **Раздел (Partition)** - логическое деление физического диска
- **Файловая система (File System)** - способ организации данных на разделе
- **Монтирование (Mount)** - процесс подключения файловой системы к иерархии каталогов
- **Точка монтирования (Mount Point)** - каталог, в который монтируется файловая система

#### Инструменты для работы с дисками
- **lsblk** - отображение информации о блочных устройствах
- **parted** - современная утилита для создания и изменения разделов (GPT/MBR)
- **mkfs** - форматирование разделов в различные файловые системы
- **mount/umount** - монтирование и размонтирование файловых систем

### Практическая часть

#### Шаг 1: Анализ доступных дисков

Перед началом работы необходимо изучить доступные диски в системе:

```bash
# Просмотр всех блочных устройств
lsblk

# Просмотр подробной информации о дисках
sudo fdisk -l

# Просмотр только дисков (не разделов)
lsblk -d
```

**Ожидаемый результат:**
```
NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
vda      8:0    0    1G  0 disk
├─vda1   8:1    0  950M  0 part /
└─vda2   8:2    0   50M  0 part /boot
vdb      8:16   0    1G  0 disk
vdc      8:32   0    1G  0 disk
vdd      8:48   0    1G  0 disk
```

В данной лабораторной работе основное внимание будет уделено диску `/dev/vdb`, который является дополнительным диском для работы с разделами и файловыми системами. Диски `/dev/vdc` и `/dev/vdd` будут использоваться в последующих лабораторных работах по темам RAID и LVM.

#### Шаг 2: Создание разделов с помощью parted

**Создание раздела с использованием GPT:**

```bash
# Запуск parted
sudo parted /dev/vdb

# В интерактивном режиме parted:
# mklabel gpt - создать GPT таблицу разделов
# mkpart primary ext4 1MB 5GB - создать первичный раздел ext4 от 1MB до 5GB
# mkpart primary xfs 5GB 10GB - создать первичный раздел xfs от 5GB до 10GB
# print - вывести информацию о разделах
# quit - выйти из parted
```

**Альтернативно, одной командой:**

```bash
# Создание GPT таблицы разделов
sudo parted /dev/vdb mklabel gpt

# Создание первого раздела
sudo parted /dev/vdb mkpart primary ext4 1MB 5GB

# Создание второго раздела
sudo parted /dev/vdb mkpart primary xfs 5GB 10GB

# Проверка результатов
sudo parted /dev/vdb print
```

#### Шаг 4: Форматирование разделов

**Форматирование в ext4:**

```bash
# Форматирование первого раздела в ext4
sudo mkfs.ext4 /dev/vdb1

# Добавление метки разделу
sudo e2label /dev/vdb1 DATA_EXT4
```

**Форматирование в xfs:**

```bash
# Форматирование второго раздела в xfs
sudo mkfs.xfs /dev/vdb2

# Добавление метки разделу
sudo xfs_admin -L DATA_XFS /dev/vdb2
```

**Форматирование в vfat:**

```bash
# Форматирование раздела в vfat (если создан отдельный раздел)
sudo mkfs.vfat /dev/vdb3

# Добавление метки разделу
sudo fatlabel /dev/vdb3 DATA_FAT
```

**Проверка созданных файловых систем:**

```bash
# Проверка всех файловых систем
lsblk -f
```

#### Шаг 5: Создание точек монтирования

```bash
# Создание каталогов для монтирования
sudo mkdir -p /mnt/data_ext4
sudo mkdir -p /mnt/data_xfs
sudo mkdir -p /mnt/data_vfat

# Установка прав доступа
sudo chmod 755 /mnt/data_ext4
sudo chmod 755 /mnt/data_xfs
sudo chmod 755 /mnt/data_vfat
```

#### Шаг 6: Временное монтирование файловых систем

```bash
# Монтирование ext4 раздела
sudo mount /dev/vdb1 /mnt/data_ext4

# Монтирование xfs раздела
sudo mount /dev/vdb2 /mnt/data_xfs

# Монтирование vfat раздела (если создан)
sudo mount /dev/vdb3 /mnt/data_vfat

# Проверка монтирования
df -h
mount | grep /mnt
```

#### Шаг 7: Проверка работоспособности файловых систем

```bash
# Создание тестовых файлов
echo "Тестовый файл для ext4" | sudo tee /mnt/data_ext4/test_ext4.txt
echo "Тестовый файл для xfs" | sudo tee /mnt/data_xfs/test_xfs.txt
echo "Тестовый файл для vfat" | sudo tee /mnt/data_vfat/test_vfat.txt

# Проверка создания файлов
sudo ls -la /mnt/data_ext4/
sudo ls -la /mnt/data_xfs/
sudo ls -la /mnt/data_vfat/

# Проверка прав доступа
sudo chmod 644 /mnt/data_ext4/test_ext4.txt
sudo chmod 644 /mnt/data_xfs/test_xfs.txt

# Проверка чтения файлов
cat /mnt/data_ext4/test_ext4.txt
cat /mnt/data_xfs/test_xfs.txt
cat /mnt/data_vfat/test_vfat.txt
```

#### Шаг 8: Постоянное монтирование через /etc/fstab

**Редактирование файла /etc/fstab:**

```bash
# Резервное копирование оригинального файла
sudo cp /etc/fstab /etc/fstab.backup

# Редактирование файла fstab
sudo nano /etc/fstab
```

**Добавление записей в /etc/fstab:**

```
# Добавьте следующие строки в конец файла /etc/fstab:

# Монтирование ext4 раздела
/dev/vdb1    /mnt/data_ext4    ext4    defaults    0    2

# Монтирование xfs раздела
/dev/vdb2    /mnt/data_xfs     xfs     defaults    0    0

# Монтирование vfat раздела
/dev/vdb3    /mnt/data_vfat    vfat    defaults    0    0
```

**Альтернативный вариант с использованием меток:**

```
# Монтирование по меткам
LABEL=DATA_EXT4    /mnt/data_ext4    ext4    defaults    0    2
LABEL=DATA_XFS     /mnt/data_xfs     xfs     defaults    0    0
LABEL=DATA_FAT     /mnt/data_vfat    vfat    defaults    0    0
```

**Монтирование по UUID (рекомендуемый способ):**

```bash
# Получение UUID разделов
sudo blkid
```

**Пример вывода:**
```
/dev/vdb1: UUID="12345678-1234-1234-1234-123456789abc" TYPE="ext4" PARTUUID="abcdef12-01"
/dev/vdb2: UUID="abcdef12-3456-7890-abcd-ef1234567890" TYPE="xfs" PARTUUID="abcdef12-02"
/dev/vdb3: UUID="98765432-1098-7654-3210-987654321098" TYPE="vfat" PARTUUID="abcdef12-03"
```

**Записи в /etc/fstab по UUID:**

```
# Монтирование по UUID (рекомендуемый способ)
UUID=12345678-1234-1234-1234-123456789abc    /mnt/data_ext4    ext4    defaults    0    2
UUID=abcdef12-3456-7890-abcd-ef1234567890    /mnt/data_xfs     xfs     defaults    0    0
UUID=98765432-1098-7654-3210-987654321098    /mnt/data_vfat    vfat    defaults    0    0
```

**Преимущества монтирования по UUID:**
- Уникальность: UUID не может повторяться
- Надежность: UUID не изменяется при переименовании меток
- Стабильность: UUID остается неизменным при изменении порядка подключения дисков

**Проверка конфигурации fstab:**

```bash
# Проверка синтаксиса файла fstab
sudo mount -a

# Проверка успешного монтирования
df -h
mount | grep /mnt
```

#### Шаг 9: Размонтирование и повторное монтирование

```bash
# Размонтирование всех разделов
sudo umount /mnt/data_ext4
sudo umount /mnt/data_xfs
sudo umount /mnt/data_vfat

# Повторное монтирование согласно fstab
sudo mount -a

# Проверка результатов
df -h
mount | grep /mnt
```

#### Шаг 10: Проверка автоматического монтирования при загрузке

```bash
# Проверка точек монтирования после перезагрузки
sudo reboot

# После перезагрузки проверьте:
df -h
mount | grep /mnt
```

## Типичные проблемы и их решения

### Проблема 1: "mount: wrong fs type, bad option, bad superblock"
**Причина:** Неправильно указана файловая система или раздел не отформатирован
**Решение:**
```bash
# Проверить тип файловой системы
sudo blkid /dev/vdb1
# Пересоздать файловую систему при необходимости
sudo mkfs.ext4 /dev/vdb1
```

### Проблема 2: "device is busy" при размонтировании
**Причина:** В каталоге монтирования открыты файлы или работает процесс
**Решение:**
```bash
# Найти процессы, использующие каталог
sudo lsof +D /mnt/data
# Завершить процессы или закрыть файлы
# Повторить размонтирование
sudo umount /mnt/data
```

### Проблема 3: Ошибки в /etc/fstab
**Причина:** Синтаксическая ошибка в файле конфигурации
**Решение:**
```bash
# Проверить синтаксис
sudo mount -a
# Исправить ошибки в файле
sudo nano /etc/fstab
# Проверить еще раз
sudo mount -a
```

### Проблема 4: Раздел не виден после создания
**Причина:** Таблица разделов не перечитана
**Решение:**
```bash
# Перечитать таблицу разделов
sudo partprobe /dev/vdb
# Или перезагрузить систему
sudo reboot
```

### Проблема 5: Недостаточно места на диске
**Причина:** Разделы занимают все доступное пространство
**Решение:**
```bash
# Проверить использование диска
df -h
# Удалить ненужные файлы или увеличить размер диска
```

## Рекомендации по безопасности

1. **Резервное копирование данных**
   - Всегда создавайте резервные копии важных данных перед работой с разделами
   - Используйте команду `dd` для создания образов дисков

2. **Проверка команд**
   - Внимательно проверяйте команды перед выполнением, особенно с параметром `-f`
   - Убедитесь, что указываете правильное устройство

3. **Работа с правами root**
   - Выполняйте команды с минимально необходимыми привилегиями
   - Не используйте `sudo` без необходимости

4. **Мониторинг системы**
   - Регулярно проверяйте состояние дисков и файловых систем
   - Используйте команды `df`, `mount`, `lsblk` для контроля

5. **Настройка резервного копирования**
   - Настройте автоматическое резервное копирование важных данных
   - Используйте cron для регулярного выполнения скриптов резервного копирования
