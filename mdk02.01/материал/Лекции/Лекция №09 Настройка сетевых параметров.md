# Практика: Настройка сетевых параметров в Alt Linux

**Оригинальная лекция:** [Лекция №09 Настройка сетевых параметров](https://github.com/aleti000/fgos2023/blob/main/mdk02.01/%D0%BC%D0%B0%D1%82%D0%B5%D1%80%D0%B8%D0%B0%D0%BB/%D0%9B%D0%B5%D0%BA%D1%86%D0%B8%D0%B8/%D0%9B%D0%B5%D0%BA%D1%86%D0%B8%D1%8F%20%E2%84%9609%20%D0%9D%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B9%D0%BA%D0%B0%20%D1%81%D0%B5%D1%82%D0%B5%D0%B2%D1%8B%D1%85%20%D0%BF%D0%B0%D1%80%D0%B0%D0%BC%D0%B5%D1%82%D1%80%D0%BE%D0%B2.md#лекция-09-настройка-сетевых-параметров)

В любой системе на базе Linux настройка сети - одно из ключевых направлений системного администрирования. От правильности сетевых параметров зависит работа служб, маршрутизация, доступ к репозиториям и возможность удалённого администрирования. В Alt Linux как и в других дистрибутивах, сетевое окружение можно управлять как вручную через консольные утилиты, так и с помощью системных сервисов.
## 📋 Основные инструменты диагностики

| Команда | Описание |
|---------|----------|
| `ip a` | Состояние интерфейсов и IP |
| `ip route` | Таблица маршрутизации |
| `ping 8.8.8.8` | Проверка доступности |
| `nmcli` | NetworkManager CLI |
| `ss -tulpn` | Активные соединения |

В более "низкоуровневом" варианте можно работать напрямую с конфигурационными файлами, где описываются параметры интерфейсов, например /etc/net/ifaces/eth0/ (в системе Alt Linux) или /etc/network/interfaces (в классическом Debian-подобном стиле). Для автоматизации настройки часто применяются DHCP-клиенты (dhclient, systemd-networkd) или Ansible-плейбуки при централизованном управлении.

Таким образом, при работе с Alt Linux администратор должен понимать не только сетевую топологию, но и структуру сетевых инструментов системы. Это позволяет осознанно выбирать подходящий метод конфигурации - будь то ручное задание параметров через ip и редактирование файлов, использование NetworkManager, или применение автоматизированных решений на базе systemd-networkd.

Далее, опираясь на эти базовые инструменты диагностики и контроля, имеет смысл перейти к систематической настройке сети в Alt Linux. В практической работе администратора это, как правило, сводится к двум уровням: оперативные изменения параметров через команды ip, nmcli и сервисные утилиты, а также постоянная конфигурация через файлы в /etc/net/ifaces и связанные системные механизмы. В следующих разделах можно последовательно разобрать типичные сценарии: просмотр и изменение параметров интерфейсов с помощью ip, создание и управление подключениями через nmcli, настройку статических и динамических адресов в каталоге /etc/net/ifaces/имя_интерфейса/, а также проверку работоспособности с использованием ping, traceroute и ss. Такой подход позволит связать теорию сетевой конфигурации с конкретными командами и файлами, которые ежедневно использует системный администратор Alt Linux.

## 🔧 Настройка через etcnet (/etc/net/ifaces/eth0/)

### 1. Основные команды
```bash
ifup eth0      # Поднять интерфейс
ifdown eth0    # Опустить интерфейс  
ifup -a        # Все интерфейсы
systemctl restart network  # Перезапуск сети
```
2. Структура файлов конфигурации

ls /etc/net/ifaces
 Показывает список интерфейсов, конфигурируемых через etcnet.

ls /etc/net/ifaces/eth0/
 Позволяет увидеть конкретные файлы конфигурации интерфейса eth0 (options, ipv4address, ipv4route, resolv.conf и т.д.).

nano /etc/net/ifaces/eth0/options
 Редактирование основного файла настроек интерфейса (тип, BOOTPROTO, ONBOOT и др.).

 nano /etc/net/ifaces/eth0/ipv4address
 Редактирование (или создание) файла с IPv4‑адресом и маской подсети.

 nano /etc/net/ifaces/eth0/ipv4route
 Задание маршрутов (обычно маршрута по умолчанию default via ...).

 nano /etc/net/ifaces/eth0/resolv.conf
 Настройка DNS‑клиента для конкретного интерфейса (nameserver, search и т.п.).

 nano /etc/sysconfig/network
 Редактирование общесистемных сетевых параметров (имя хоста, домен и пр.).

 sysctl -a | grep net.
 Просмотр всех сетевых параметров ядра, применяемых через sysctl.

 nano /etc/net/sysctl.conf
 Задание сетевых параметров ядра, которые будут применяться при старте сети etcnet.
 
```bash
options
Базовые параметры интерфейса: тип, способ получения адреса, автозапуск.
(статический IP):
TYPE=eth
DISABLED=no
NM_CONTROLLED=no
CONFIG_IPV4=yes
ONBOOT=yes

ipv4address
Указывает IPv4‑адрес и маску подсети для интерфейса при статической настройке:
192.168.0.2/24

ipv4route
Задаёт маршруты для данного интерфейса, в том числе маршрут по умолчанию.
default via 192.168.0.1
Можно добавлять дополнительные сети:
10.0.0.0/24 via 192.168.0.254

resolv.conf
Локальный DNS для интерфейса, если хочешь привязать свои DNS/домены именно к этому iface.
search mydom1.local domain2.ru
nameserver 8.8.8.8
nameserver 192.168.0.5
```

Общесистемные файлы

/etc/sysconfig/network
Хранит общие сетевые параметры: имя хоста, доменное имя и пр.
Типичный минимум:
HOSTNAME=server1.mydom1.local
NETWORKING=yes

/etc/net/sysctl.conf
Дополнительные параметры ядра, которые применяются при старте сети (аналог /etc/sysctl.conf, но заточен под etcnet).
Пример «классики» для маршрутизатора:
net.ipv4.ip_forward = 1
net.ipv4.conf.all.rp_filter = 1
net.ipv4.conf.default.accept_source_route = 0

## Минимальный сценарий статической настройки
    1. Создать каталог интерфейса: mkdir -p /etc/net/ifaces/eth0. 
    2. Заполнить options с CONFIG_IPV4=yes и ONBOOT=yes. 
    3. Записать адрес в ipv4address (например, 192.168.0.2/24). 
    4. Прописать шлюз в ipv4route (например, default via 192.168.0.1). 
    5. При необходимости задать DNS в resolv.conf. 
    6. Перезапустить сеть: systemctl restart network или ifdown eth0 && ifup eth0.
Дополнительные возможности etcnet
    • Несколько профилей на один интерфейс: можно завести несколько подкаталогов (например, eth0-home, eth0-office) и переключать их через свои юниты network@profile.service. 
    
    • IPv6‑конфигурация: аналогично IPv4 можно использовать файлы ipv6address, ipv6route с адресами в формате 2001:db8::1/64 и маршрутами.
    
    • Тонкая настройка DNS: разные resolv.conf для разных интерфейсов позволяют разруливать сложные схемы с несколькими провайдерами/сетями. 
    
    • Сетевые sysctl‑тюнинги: через /etc/net/sysctl.conf удобно задавать параметры ядра, связанные с маршрутизацией, фильтрацией и производительностью, чтобы они гарантированно применялись при поднятии сети.
    
3. Минимальная статическая настройка
```bash
mkdir -p /etc/net/ifaces/eth0
# Заполнить options, ipv4address, ipv4route
ifdown eth0 && ifup eth0
```

## ⚡ Настройка через nmcli

### 1. Диагностика состояния
```bash
nmcli connection show     # Все подключения NetworkManager
nmcli device status       # Состояние интерфейсов
```

### 2. Статический IP (полная команда)
nmcli connection add type ethernet ifname eth0 con-name eth0-static \
ipv4.method manual \
ipv4.addresses 192.168.0.2/24 \
ipv4.gateway 192.168.0.1 \
ipv4.dns "8.8.8.8 192.168.0.2" \
connection.autoconnect yes

Создаёт профиль eth0-static с полным набором параметров IPv4 и автозапуском.

### 3. Управление подключением
nmcli connection show eth0-static    # Проверка профиля
nmcli connection up eth0-static      # Активация
nmcli connection down eth0-static    # Деактивация
nmcli connection delete eth0-static  # Удаление

### 4. DHCP-подключение
nmcli connection add type ethernet ifname eth0 con-name eth0-dhcp \
ipv4.method auto connection.autoconnect yes
nmcli connection up eth0-dhcp

### 5. Проверка результата
ip a show dev eth0                    # IP-адрес интерфейса
ip route                              # Таблица маршрутизации
ping -c3 192.168.0.1 && ping -c3 8.8.8.8 && ping -c3 ya.ru

### 6. Быстрое переключение профилей
nmcli connection down eth0-static
nmcli connection up eth0-dhcp

### 7. Основные команды управления
| Действие        | Команда                                                              |
| --------------- | -------------------------------------------------------------------- |
| Список активных | nmcli connection show --active                                       |
| Изменить IP     | nmcli connection modify eth0-static ipv4.addresses "192.168.1.10/24" |
| Перезагрузить   | nmcli connection up eth0-static                                      |

🚀 Рабочий цикл администратора

1. nmcli device status     ← что подключено?
2. nmcli connection show   ← какие профили есть?
3. nmcli connection add    ← создать/изменить
4. nmcli connection up     ← активировать
5. ip a + ping             ← проверить
6. nmcli connection down   ← переключиться

Преимущество nmcli: атомарные команды, без редактирования файлов, легко в скрипты и Ansible.
